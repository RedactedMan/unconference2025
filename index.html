Tips for Building Websites with AI Prompts

üí¨ Clear, Direct Commands

Be specific and direct about what you want. AI responds well to clear instructions.

Examples from this presentation:

"Update the last slide with the information in the attached email."
"Add links to the Technology Stack list to websites for the listed tools."
"Change the last 2 slides to not use bullet points. Try another way to present this information."
"Add a copy to clipboard button on 'Your Turn: Try This Prompt' to make it easy to get the contents of the page out."

üíæ Commit Changes You Like

Use version control (like Git) to save your progress. Sometimes you need to revert to an older version if a new prompt removes features you like or breaks something that was working.

Pro Tip: After each successful change you're happy with, commit it. This gives you safe restore points.

üéØ Feel Free to Make Big and Small Prompts

Don't be afraid to request both radical changes and tiny tweaks. AI can handle:

Big changes: "Redesign the entire layout" or "Add a new slide with..."
Small changes: "Fix the spacing on that button" or "Change the color to blue"
Recovery: If something goes wrong, you can always back it out with version control or ask the AI to revert the change

üîÑ Iterative Refinement

Building with AI is an iterative process. Start with a basic request, see the result, then refine. Each prompt builds on the previous work.

‚ùì Ask Questions for Ideas

Don't just give commands‚Äîask the AI for suggestions on next steps, style improvements, or functionality enhancements. The AI can often see opportunities you might miss.

Examples from this session:

"Do you have suggestions on how to make the presentation more visually appealing?"
"Do you think judicious use of stock images could make the presentation look better?"
"How does this page look on mobile compared to a desktop? Are there any changes that would make it more mobile friendly?"
